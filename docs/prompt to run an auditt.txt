MyGang Pre-Production Deep Audit & Improvement Plan

    ## Project Context

    MyGang is a social AI chat app where users talk to a "gang" of AI characters (like a group chat). Built with
  Next.js
    16, Supabase, TypeScript, Tailwind CSS, and deployed on Vercel.

    ### Tech Stack
    - **Framework**: Next.js 16 (App Router) — uses `src/proxy.ts` for middleware, NOT `middleware.ts`
    - **Database**: Supabase (Postgres + RLS + Auth)
    - **Auth**: Supabase Auth via `@supabase/ssr`, auth-first flow (no guest mode)
    - **Billing**: DodoPayments (webhooks for subscription lifecycle)
    - **Rate Limiting**: Upstash Redis (sliding window, fail-closed in production)
    - **AI**: OpenRouter API (via `ai` SDK's `generateObject`)
    - **State**: Zustand (`chat-store.ts`) with localStorage persistence
    - **Package Manager**: pnpm (NEVER npm)
    - **Deployment**: Vercel, all env vars stored there

    ### Tier System (src/lib/billing.ts)
    - **Free**: 20 messages per 1-hour sliding window, no memory, 4 squad members, 10 context messages
    - **Basic** ($14.99/mo): 500 messages/month (falls back to free-tier hourly window when exhausted), memory
  enabled, 5
    squad members, 20 context messages
    - **Pro** ($19.99/mo): Unlimited messages, memory enabled, 6 squad members, 30 context messages

    ### Key Architecture Files
    - `src/app/api/chat/route.ts` — Main chat API (~1500 lines). Handles: auth, rate limiting, paywall, LLM calls
  via
    generateObject, memory retrieval/storage, chat history persistence, autonomous continuation
    - `src/hooks/use-chat-api.ts` — Client-side chat hook. Message queuing, delivery status, paywall detection,
  autonomous
     flow, typing simulation
    - `src/hooks/use-autonomous-flow.ts` — Idle banter, autonomous continuation logic
    - `src/lib/rate-limit.ts` — Upstash Redis rate limiter, in-memory fallback for dev, fail-closed in prod without
   Redis
    - `src/lib/ai/memory.ts` — Memory system: store, retrieve (with embeddings), compact, touch
    - `src/lib/ai/openrouter.ts` — OpenRouter model configuration
    - `src/stores/chat-store.ts` — Zustand store for chat state, messages, gang, settings
    - `src/proxy.ts` — Auth guards for /chat, /onboarding, /settings, /checkout/success
    - `src/app/api/webhook/dodo-payments/route.ts` — Billing webhooks (idempotent, error-checked)
    - `src/app/api/checkout/route.ts` — Checkout session creation
    - `src/app/api/checkout/activate/route.ts` — Post-checkout activation with ownership verification
    - `src/lib/billing.ts` — Tier definitions, helper functions
    - `src/constants/characters.ts` — Character definitions (personalities, traits, genders)
    - `src/components/chat/` — All chat UI components (message-item, chat-settings, memory-vault, etc.)
    - `src/components/settings/settings-panel.tsx` — Settings/control center page
    - `src/app/auth/actions.ts` — Server actions (deleteAccount, deleteAllMessages, deleteAllMemories, signOut,
  settings,
    memory CRUD)
    - `src/lib/supabase/admin.ts` — Admin Supabase client (service role)
    - `src/lib/supabase/server.ts` — Server-side Supabase client
    - `src/lib/supabase/client.ts` — Browser Supabase client
